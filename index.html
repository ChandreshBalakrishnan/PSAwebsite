Prototype System Overview (Folder 04_Prototype)

This prototype exemplifies a single module of an embedded greenhouse lighting system, modelled using the finite state machine (FSM) in Figure 1.

Software Subsystem (Folder 04.2.2_Software)

The software subsystem is implemented in CircuitPython on the Raspberry Pi Pico and realizes the system’s control logic based on a finite state machine (FSM) model, shown below.

The FSM defines three operating states: OFF (all LEDs off), PLANTS (red and blue LEDs on), and PLANTS & PEOPLE (all LEDs on). System behavior is controlled through two pushbutton inputs, Forward and Backward, which cyclically advance or reverse the system state. The Reset input indicates the system’s initial state. Folder 04.2.2_Software contains the iterative code development, including the intermediate and final code versions with their corresponding FSM representations.

Electrical Subsystem

The electrical subsystem consists of a Raspberry Pi Pico, LEDs, resistors, and two pushbuttons on a breadboard. Resistors were selected to prevent LED damage, cross-referencing Lab 2 and observation of brightness through trials. The final circuitry design was selected to minimize the use of wires, and keep LEDs centered design for cleanliness, aesthetics, and reproducibility.

Structural Subsystem

The final structural subsystem consists of a digitally fabricated enclosure designed to protect the electrical components and improve aesthetic appeal. PLA was selected due to MYFab fabrication constraints. The enclosure design prioritized accessibility to the pushbutton and USB port while minimizing material usage and print time. Screws were used to secure the enclosure to design for durability. Further iterations can be found in Folder 04.2.2_Structural.
