Software Diverging and Converging

The software subsystem followed a similar diverging and converging methodology, centered on interaction logic and state accessibility. The SCAMPER analysis for software documents the evolution from simple, role-specific button behaviors toward a fully state-based control strategy.

Divergence occurred by substituting one-directional state cycling with bidirectional navigation, replacing mode-specific logic with explicit finite state machine (FSM) modeling, and exploring different mappings between button inputs and LED modes. Combination and adaptation incorporated principles from user interface design, embedded systems modeling, and consumer electronics interaction patterns. Modifications focused on improving state reachability, symmetry, and predictability, while elimination removed dead-end states and redundant logic that limited usability.

Convergence was achieved through a pairwise comparison of the three software prototypes, each represented by both code and an FSM. These comparisons evaluated state accessibility, user intuitiveness, and alignment with the finalized two-button electrical interface. The final software design was selected based on its ability to provide complete, symmetric state access while remaining simple to understand, document, and extend.
